# Knowledge Base Categories

## Dev
**Definition:** Technical knowledge related to software development, engineering, and tools.

**Classify as Dev when content mentions:**
- Programming languages, frameworks, libraries
- Code patterns, algorithms, data structures
- Debugging techniques, error resolution
- Architecture decisions, system design
- DevOps, deployment, CI/CD, infrastructure
- APIs, databases, tools, IDEs
- Git, version control, code reviews

**Example entries:**
- "How to set up ESLint with TypeScript"
- "Postgres query optimization notes"
- "Architecture decision: microservices vs monolith"

## Research
**Definition:** Learning material, articles, courses, and conceptual knowledge.

**Classify as Research when content mentions:**
- Articles, blog posts, papers
- Book notes, course takeaways
- Tutorials, how-tos (non-code-specific)
- Concepts, frameworks, methodologies
- Industry trends, analysis
- Ideas being explored

**Example entries:**
- "Summary of 'Thinking in Systems' by Donella Meadows"
- "Notes on RAG architecture patterns"
- "Interesting approach to team retrospectives"

## Meetings
**Definition:** Meeting notes, decisions made in discussions, and action items.

**Classify as Meetings when content mentions:**
- Meeting notes, standups, retros
- Stakeholder discussions, decisions
- Action items, follow-ups
- Project updates, status reports
- 1:1 notes, team sync notes

**Example entries:**
- "Sprint planning - Feb 10"
- "1:1 with manager - career goals discussion"
- "Architecture review meeting notes"

## Personal
**Definition:** Personal goals, reflections, habits, and non-work knowledge.

**Classify as Personal when content mentions:**
- Personal goals, resolutions
- Journal entries, reflections
- Health, fitness, habits
- Personal projects (non-work)
- Life admin, finance notes
- Hobbies, interests

**Example entries:**
- "2024 goals review"
- "Workout routine notes"
- "Home automation project ideas"
